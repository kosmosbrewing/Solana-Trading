## 2. "Dead Liquidity" 가설 — 12h 데이터 기반 팩트체크

### 2.1 4가지 증거

**증거 1: V2 PASS 3180 건이 오직 2 pair 에 집중**
- pair A: `4aubkZ8LtE2W...` (낯선)
- pair B: `8CyoUy31zfTc...` (낯선)
- ASTEROID / MAGA / BELIEF / BULL 은 V2 PASS 리스트에 단 한 번도 없음
- → pump-and-dump 봇이 같은 pair 반복 churn 하는 패턴

**증거 2: Survival Gate 통과율 = 2%**
- V2 PASS 3180 중 63 통과, 3117 이 holder concentration > 80% 로 reject
- 진짜 살아있는 mid-cap memecoin 은 보통 top 10 holder 30-50%
- → scan 대상 pool 자체가 rug-prone 다수

**증거 3: Signal price bug `deltaPct p50 = −92%` 의 진짜 의미**
- signal price: 0.00349601 SOL / Jupiter actual: 0.00027854 SOL (12배 gap)
- 이건 **코드 버그 아닐 수 있음** — stale pool 의 last swap 가격이 남아있지만 실제 유동성 없음
- Jupiter 가 실 route 찾으면 훨씬 낮은 가격 → drift reject
- → stale pool 을 "burst 중" 으로 오인해서 V2 detector 가 PASS

**증거 4: ASTEROID / MAGA / BELIEF / BULL 은 Scanner watchlist 에 있었지만 V2 PASS 0건**
- 9h 로그: `Scanner: new candidate ASTEROID (30회)` 등 watchlist 진입 확인
- 그러나 V2 detector 가 이들에서 burst 를 못 찾음
- → 이미 대형 유동성 시장. 우리 detector 의 "sub-second 폭발" 패턴이 안 나옴 (정상적 trend 이므로 z-score 기반 burst 감지 불가)

### 2.2 결론

> **우리가 scan 하는 pipeline 은 "방금 생긴 pool 의 첫 1분 burst" 만 볼 수 있고, 이건 구조적으로 "dead wash-trade + rug-prone" 에 편향된다. 살아있는 시장 (ASTEROID 등) 은 우리 pipeline 이 구조적으로 감지 못 한다.**

### 2.3 왜 ASTEROID 를 못 잡았는가

```
Phase 1: Initial pump       (0-30s)    +3% ~ +20%
Phase 2: Consolidation      (30s-3min) -5% ~ +5% 횡보
Phase 3: Breakout           (3-30min)  +50% ~ +500% 🔥
Phase 4: Trend continuation (30min+)   10x ~ 100x 🚀
```

ASTEROID 급은 **Phase 3-4 에서 본체**. 우리 `pure_ws`:
- Phase 1 만 먹는 구조 (30s PROBE, T1 +100% 도달 전 82% 사망)
- Phase 2 consolidation 을 "burst 실패" 로 오판
- Phase 3 는 이미 포지션 없음

## 3. Information Flow 분석 — Alpha 가 어디서 나오는가

### 3.1 Memecoin 시장의 정보 흐름

```
T-30분: Twitter / Telegram 에서 narrative 시작
T-15분: 주요 influencer retweet
T-5분:  Smart wallet 들이 조용히 accumulate
T-0:    Retail FOMO 시작 → on-chain volume 폭발
T+0s:   우리 V2 detector 가 드디어 감지
T+5-10s: Signal fire
T+10s:  Entry
T+30s:  Phase 2 consolidation → 우리 봇 cut
T+3-30min: Phase 3 breakout (10x~100x)
```

**→ 우리는 정보가 완전히 공개된 후 마지막 조각을 보고 뛰어드는 구조 = late alpha**

|------|------|------|----------|
| Narrative | T-30~T-5min | Twitter, Telegram, Discord | ❌ |

## 5. 🎯 핵심 아이디어 — Trending-Gated Scalping Bot

사용자 제안: *"이걸로 트렌딩을 찾고 나오는 변동성에서 스캘핑 봇을 만든다면?"*

### 5.1 2 Layer 분리

```
Layer 1 (Discovery): "이 pair 살아있는가?"  ← Trending / Graduate WS 가 답함
                      ↓
Layer 2 (Execution): "지금 변동성 파동 하나 잡자"  ← Scalping 로직이 답함
```

- 현재 Layer 1 부재 → 죽은 pool 에서 scalp 시도 → 반드시 망함
- Layer 1 붙이면 현 pure_ws 구조가 **드디어 의미 있어짐**

### 5.2 구조 제안

```
┌─────────────────────────────────────────────────────┐
│ Layer 1: Discovery (신규)                            │
│  ├─ DexScreener Boost / Trending poller (60s cycle) │
│  ├─ Pump.fun Graduate WS (real-time)                │
│  └─ GeckoTerminal Hot Pool (300s cycle)             │
│       ↓                                              │
│  [Universe Filter]                                   │
│   - volume_24h > $50k                                │
│   - liquidity > $20k                                 │
│   - age 1h ~ 7d (너무 신선 = rug, 너무 오래 = 식음)  │
│   - holder concentration < 60%                       │
│   - sell quote probe pass (exitability)              │
│       ↓                                              │
│  [Active Universe]  ← 300~500 pair                   │
└─────────────────────────────────────────────────────┘
                     ↓
┌─────────────────────────────────────────────────────┐
│ Layer 2: Scalping Execution (현 pure_ws 재활용)      │
│  ├─ 이 universe 안에서만 V2 detector 돌림            │
│  ├─ 5m ~ 15m 변동성 파동 감지                        │
│  ├─ Entry: 2-5% dip after uptrend                   │
│  ├─ Target: +3~10% scalp                            │
│  ├─ Stop: -2%                                       │
│  └─ Hold: max 30 min                                │
└─────────────────────────────────────────────────────┘
```

### 5.3 현 pure_ws 대비 차이

| 항목 | 현 pure_ws | Trending scalping |
|------|-----------|-------------------|
| Discovery | Helius pool discovery 무조건 | Trending API 통과 only |
| Universe 크기 | 전 세계 새 pool | 300-500 "살아있는" pair |
| Detection window | 60s sub-second burst | 5-15min swing |
| Entry trigger | z-score 폭발 | dip-after-uptrend |
| Target | T1 +100% (never reached) | +3-10% (realistic) |
| Hold | 30s | 5-30min |
| Trade 빈도 | 낮음 (12h 에 6 live buy) | 높음 (expected 20-50/day) |

### 5.4 State Machine (현 pure_ws 보다 단순)

```
[SIGNAL] → loose filter → [ENTRY]
    ↓
[SCALP]  5min window, trail 1.5%, hardcut -2%
  ├─ MFE ≥ +3% → [LOCK] 1.5% trail, 최소 +1%
  │              → WINNER_TRAILING exit
  ├─ MAE ≤ -2% → LOSER_HARDCUT
  └─ 5min 만료 → FLAT_EXIT (-1% ~ +1% close)
```

**T1/T2/T3 promotion 없음**. Scalping 은 단일 phase.

## 6. 📊 기대값 수학 — 정말 남는가?

### 가정 (보수적)
- Trending universe 에서 daily volatility 평균 **±15%**
- Scalp 목표 = **+3%** (target) / **-2%** (stop)
- Win rate = **55%** (trending = 추세 존재)
- Fee + slippage = **round-trip 0.5%** (ticket 0.01 SOL)

### Per-trade expectancy

```
E[trade] = 0.55 × 3% - 0.45 × 2% - 0.5%
        = 1.65% - 0.90% - 0.5%
        = +0.25% per trade
```

Ticket 0.01 SOL → **+0.000025 SOL / trade**

- 30 trades/day × 30 days = 900 trades
- 900 × 0.000025 SOL = **+0.0225 SOL / month**

### 현실 체크

- 기대 이익 +0.0225 SOL/month vs 현 bleed −0.0019 SOL / 12h (= −0.114 SOL/month)
- **상대 개선**: +0.14 SOL/month (= 15% floor 회복)
- **절대값**: 크지 않음. 0.9 SOL → 1.0 SOL 도달도 6개월+

### 사명 관점

- **"0.8 floor 유지"**: ✅ 매우 안전
- **"200 trades"**: ✅ 과도하게 달성 (900/month)
- **"5x+ winner 실측"**: 🔴 구조적 불가 (target 3%)

→ **사명 그대로 유지 시 scalping 은 safety lane (tail hunter 는 별도)** 또는 **hybrid** 필수.

## 7. ⚠️ 함정 2개

### 함정 1: "Trending" 진입 직후 = Phase 3-4 = reversal 대기 중

```
Trending 진입의 의미:
  "이 토큰은 이미 뉴스가 퍼졌고 FOMO 중"
  = pump 후반부
  = 곧 reversal
```

→ scalping long-only 는 top 에서 sell 에 당할 수 있음.
완화책:
- Entry 시점 세분화: trending 진입 **직후 1시간 내만** entry 허용
- Volume trend (증가 vs 감소) 조건 추가
- 또는 short-also 지원 (Solana 환경에서 어려움)

### 함정 2: 수수료가 scalping 의 천적

Solana swap cost (Jupiter):
- Platform fee: 0.1%
- AMM fee: 0.2-0.3%
- MEV slippage: 0.1-0.3%
- Priority fee: 0.001-0.01 SOL / tx

Round-trip: **0.5-0.8%**

Target 3% 에서 0.8% 가 사라지면:
```
E' = 0.55 × (3% - 0.8%) - 0.45 × 2%
   = 1.21% - 0.9% = +0.31%  (여전히 양)
```

Target 을 **+2%** 로 낮추면:
```
E' = 0.55 × (2% - 0.8%) - 0.45 × 2%
   = 0.66% - 0.9% = -0.24%  (실패)
```

→ **Target 최소 3% 이상 유지** + Priority fee 최소화 + direct AMM routing 검토.

## 8. ⚖️ 사명 재해석 3옵션 (운영자 결정 필요)

현 사명:
```
0.8 SOL floor 유지 + 200 trades + 5x+ winner 실측
```

Scalping 도입은 "5x+ winner" 정의와 충돌. 재해석 3옵션:

### 옵션 A. 사명 그대로 + Scalping 은 별도 lane
- Scalping bot = **생존 수단 + safety net**
- Pure_ws / 신규 tail lane = **여전히 5x+ winner 사냥**
- 2 lane 병행 운영. **가장 보수적**, mission-refinement 원문 준수.

### 옵션 B. 사명 update (positive wallet log growth)
- "5x+ winner" = 5x 수익률 달성 **방법** (단발 vs 누적 무관)
- Scalping 으로 wallet 5x (0.9 → 4.5 SOL) 도 사명 달성
- mission-refinement §2.3 "기술적 성공" 재정의 — 운영자 결정

### 옵션 C. 하이브리드 (본 문서 추천)
- Scalping lane = **주 수익원 + 생존 + trade count 기여**
- Tail hunter lane (신규) = **사명의 "5x+ winner 실측"** 담당, paper-first
- Stage 2/3 판정은 **lane 별** 분리:
  - Scalping: "positive wallet log growth"
  - Tail: "5x+ winner count"

## 13. 💬 핵심 통찰 정리

1. **Helius 는 "무엇이 일어났는가" 의 녹화**. Alpha 는 "누가 왜" 에서 나옴.
2. **memecoin alpha 의 80% 는 Narrative + Smart Wallet**. 나머지 20% 만 순수 on-chain.

4. **ASTEROID / MAGA 를 놓친 건 버그가 아니라 설계 의도대로의 결과**.
5. **"Trending-gated scalping"** = 죽은 유동성 문제를 해결하는 가장 현실적이고 사명 친화적인 paradigm.
6. **Scalping ≠ Tail hunter**. 이걸 혼동하면 사명을 배반함.
7. **Helius 안에서 고민하지 말고, 밖으로 나가는 게 맞음** — 단, 방어선 (Real Asset Guard) 유지 조건.
