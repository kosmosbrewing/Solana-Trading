# Core Beliefs — 에이전트 우선 운영 원칙

## 봇 원칙
1. **이유 없이 추격하지 않는다** — 설명 불가 급등은 조작 가능성으로 간주
2. **이벤트가 확인된 코인만 감시한다** — AttentionScore가 선행 컨텍스트
3. **온체인 확인 후에만 진입한다** — 브레이크아웃은 트리거일 뿐
4. **모든 트레이드를 추적 가능하게 기록한다** — source attribution 필수

## 에이전트 운영 원칙
1. **레포가 유일한 진실의 원천** — 외부 채널 결정은 레포에 커밋하기 전까지 존재하지 않음
2. **규칙은 기계적으로 강제** — markdown에 적는 건 제안, CI에서 실패시키는 건 규칙
3. **지루한 기술이 이긴다** — 에이전트 학습 데이터에 풍부한 기술 우선

## 에이전트 실패 시 대응 절차 (Feedback Loop)

에이전트가 잘못된 코드를 생성했을 때:

1. ❌ 프롬프트만 고치지 마라.
2. ❌ 사람이 직접 코드를 수정하지 마라.
3. ✅ "어떤 컨텍스트가 누락됐는가?"를 파악하라.
4. ✅ 누락된 것을 레포에 추가하라:
   - API 패턴이 틀렸다면 → ADR 또는 관련 docs/ 문서 보강
   - 의존성 방향을 위반했다면 → ESLint 규칙 추가
   - 비즈니스 로직을 오해했다면 → `docs/design-docs/` 문서 보강
   - 컨벤션을 어겼다면 → `../CONVENTIONS.md` 규칙 추가
5. ✅ 그 수정도 에이전트가 작성하게 하라.

> 이 루프가 하네스를 시간이 갈수록 강화시키는 핵심 메커니즘이다.
