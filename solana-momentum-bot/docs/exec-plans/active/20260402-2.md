## 배경

### 현재 병목

MomentumTrigger는 3개 AND 조건을 요구:
1. Volume surge: `currentVolume / avgVolume >= 2.5`
2. Price breakout: `close > highestHigh(20봉)`
3. Confirmation: 60s 캔들 3봉 연속 양봉 + 2% 이상 변동

**RejectStats (vm=2.5 시점):**
```
evals=7 signals=0 | insuffCandles=22 volInsuf=6 noBreakout=7 confirmFail=7
```

- volInsuf: 6/7 (86%) — volume gate 여전히 지배적
- noBreakout: 7/7 (100%) — 20봉 신고가 돌파 전무
- confirmFail: 7/7 (100%) — confirm 전부 실패

### 근본 원인

밈코인 모멘텀은 1~2봉(10~20초)이면 끝남. 3봉 confirm(30초+)을 기다리면 이미 늦음.
breakout은 시장 전체가 조용하면 구조적으로 불가능.
→ **trigger 조건이 과잉, 데이터 소스나 전략이 아닌 판정 로직이 병목**

### Codex 분석 (OnchainAnalyst)

**Dual-lane 제안:**
- **Core lane**: 설명 가능한 후보 → 온체인 확인 → 진입 (장기 레일, PROJECT.md 기준)
- **Bootstrap lane**: Helius WS 기반 volume explosion 감지 → 50 trades 빠르게 확보

**최소 안전장치:**
- SOL quote
- Security gate
- Execution viability
- 기본 context/source attribution
- Reject telemetry

**운영 순서:**
1. Trigger telemetry 확보 → ✅ 완료 (RejectStats 배포)
2. Bootstrap lane으로 signal cadence 확보
3. Live 50 trades 적재 (PnL보다 cadence/execution/reject mix 우선)
4. 그 결과로 core lane 재조정

### Mission Gate 정합성 검증

| Mission Gate 조건 | Bootstrap lane 충족 |
|---|---|
| 설명된 진입 비율 ≥ 90% | ✅ watchlist source attribution 유지 |
| Source attribution missing = 0 | ✅ trending/boost 경로 추적 가능 |
| Safety bypass = 0 | ✅ security gate + exec viability 유지 |

**핵심**: Watchlist 내 토큰만 대상. volume explosion은 Stage 2(trigger)를 단순화하는 것이지 Stage 1(context)을 생략하는 게 아님.

### Step 1: VolumeMcapSpikeTrigger (신규 파일)

**위치**: `src/strategy/volumeMcapSpikeTrigger.ts`

**Trigger 로직:**

```
onCandle(candle, candleBuilder) → Signal | null

1. interval 필터: primaryIntervalSec(10s)만 처리
2. lookback 체크: candles < lookback+1 → insufficientCandles++
3. Volume acceleration (HARD gate):
   volumeRatio = current.volume / calcAvgVolume(previous, lookback)
   volumeRatio < multiplier → volumeInsufficient++, return null
4. Buy ratio (SOFT gate):
   buyRatio = calcBuyRatio(current)  ← breakoutScore.ts 재사용
   buyRatio < minBuyRatio → lowBuyRatio++, return null
5. Cooldown 체크
6. Signal emit (strategy: 'volume_spike', meta.realtimeSignal: 1)
```

**MomentumTrigger와 차이:**

| | MomentumTrigger (core) | VolumeMcapSpikeTrigger (bootstrap) |
|---|---|---|
| Volume surge | ✅ | ✅ (동일) |
| 20봉 breakout | ✅ 필수 | ❌ 제거 |
| 3봉 confirm | ✅ 필수 | ❌ 제거 |
| Buy ratio | ❌ 없음 | ✅ 0.55 soft filter |
| mcap enrichment | ❌ | ✅ meta.volumeMcapPct |

**RejectStats (5분 주기):**
```
[VolumeMcapSpike] [RejectStats] evals=N signals=N | insuffCandles=N volInsuf=N lowBuyRatio=N cooldown=N
```

**Signal 출력:**
```typescript
{
  action: 'BUY',
  strategy: 'volume_spike',     // gate chain 호환 유지
  pairAddress, price, timestamp,
  meta: {
    realtimeSignal: 1,
    triggerMode: 1,              // 1=bootstrap (Record<string,number>)
    primaryIntervalSec, primaryCandleStartSec, primaryCandleCloseSec,
    volumeRatio, avgVolume, currentVolume,
    buyRatio, atr,
    volumeMcapPct?,              // mcap 있을 때만
  }
}
```

**재사용하는 기존 코드:**
- `calcATR`, `calcAvgVolume` ← `src/strategy/indicators.ts`
- `calcBuyRatio` ← `src/strategy/breakoutScore.ts`
- `buildMomentumTriggerOrder` ← `src/strategy/momentumTrigger.ts`
