```
1 → 2 SOL   Bootstrap (1% risk, ~70 trades)
2 → 5 SOL   Calibration (1% risk, ~90 trades)
5 → 20 SOL  Confirmed + Kelly 3% (~50 trades)
20 → 100 SOL Proven + Kelly 5% (~35 trades)
             총 ~245 trades, 하루 5건이면 ~7주
```

핵심: Bootstrap 50 trades를 빠르게 돌파해야 복리 구간에 진입한다.
하루 5건 전제 → 10일. 하루 2건이면 25일. **concurrent + signal 밀도가 속도를 결정한다.**

## Phase 1: Signal 밀도 확보 (1주차)

### 1A. VPS 최신 코드 배포 ⚡ (즉시)

- `git pull && npm ci && tsc && pm2 restart` — Meteora whitelist 반영
- 검증: runtime-diagnostics에서 `unsupported_dex` 0건 확인
- 추가: `meteoradbc` → `isMeteoraDexId()` 추가 (4건이지만 누락 방지)
- 예상 효과: 후보풀 698/825 = **85% 복원**

### 1B. Age Bucket 완화 ⚡ (즉시)

- `AGE_BUCKET_HARD_FLOOR_MIN` 15 → **5** (분)
- 근거: backtest에서 밈코인 MFE peak이 5~10분 구간에 집중
- 15min floor → 최적 모멘텀 윈도우 전체 차단 중
- 리스크 완화: 5min 미만만 reject, 5~15min은 기존 tier multiplier(0.25x)로 사이징 축소
- 예상 효과: 진입 가능 토큰 **50~70% 증가**

### 1C. Concurrent 확대 (즉시)

- `MAX_CONCURRENT_POSITIONS` 1 → **2**
- Bootstrap equity <5 SOL이지만, 1% risk × 2 positions = 2% 동시 노출 → 안전 범위
- `MAX_CONCURRENT_ABSOLUTE` 3 유지 (hard cap)
- 예상 효과: 일일 trade 가능량 **2배**

### 1D. Watchlist 확대

- `MAX_WATCHLIST_SIZE` 8 → 20 (config.ts 기본값 이미 20)
- `REALTIME_MAX_SUBSCRIPTIONS` 8 → 30 (config.ts 기본값 이미 30)
- Gecko 429 추이 관찰 필요 — 증가 시 `SCANNER_DEX_ENRICH_MS` 조정
- 예상 효과: 동시 추적 페어 2.5배 증가

### 1E. Trigger 파라미터 관찰

- 1A~1D 적용 후 24~48h 관찰 (vm=3.0, cb=3, cp=2%)
- signal 0 지속 시 `REALTIME_CONFIRM_MIN_BARS` 3 → 2 완화
- micro-backtest 근거: cb=2가 신호 밀도 2배 이상 (16 vs 34 signals)
- vm=3.0 → 2.5 완화는 1E 관찰 결과에 따라 결정
