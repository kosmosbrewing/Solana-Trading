### Signal 0 원인 분해

pm2 로그 100,300줄에서 eval/trigger/signal 관련 로그 = **0줄**.
이는 "평가가 실행되지 않았다"가 아니라 **"평가 결과가 로그에 노출되지 않았다"**의 증거.

코드 내부에 `rejectStats` 카운터가 존재하고 매 10초 캔들마다 `onCandle()`이 호출되지만,
runtime은 이 stats를 pm2 로그나 runtime-diagnostics로 내보내지 않는다.

**0 signal은 단일 원인이 아닌 복합 병목:**

```
① Admission 병목: 6 active pairs, queue 12,483 drop → 평가 대상 자체가 적음
② 보수적 trigger 파라미터: vm=3.0, cb=3, cp=2% → 6개 페어로는 조건 충족 확률 낮음
③ Trigger telemetry 부재 → ①과 ②의 기여도를 운영 로그만으로는 분리 불가
```

**타이밍:** 첫 신호 가능 시점 = `max(21×10s, 3×60s)` = **210초 ≈ 3분 30초** (lookback과 confirmation은 병렬 충족)

### 수정 사항

- `momentumTrigger.ts`: `createModuleLogger` 추가, 5분 주기 `[RejectStats]` INFO 로그
- `momentumTrigger.ts`: volume 또는 breakout 하나 이상 통과 시 `Near-miss` DEBUG 로그
- `index.ts`: signal 발생 시 INFO 로그 추가

배포 후 5분마다 아래와 같은 로그가 출력됨:
```
[MomentumTrigger] [RejectStats] evals=342 signals=0 | insuffCandles=1205 volInsuf=340 noBreakout=330 confirmFail=342 cooldown=0
```

이 데이터로 어떤 조건이 가장 큰 병목인지 정량적으로 파악 가능.

## Preflight Fallback 분석

### 메커니즘

```
TRADING_MODE=live
  → src/index.ts:249 preflight 실행
  → 실패 시 + PREFLIGHT_ENFORCE_GATE=true
  → effectiveMode='paper' (index.ts:262)
  → 세션 디렉터리: ${timestamp}-paper (persistenceLayout.ts:47)
  → current-session.json에 paper 기록
```

`current-session.json`은 마지막 세션 포인터 — 이전 live 세션이 있어도 이후 paper 세션이 시작되면 덮어씌워진다. `sessions/` 아래의 live 디렉터리들이 그 증거.

### VPS 로그에서 확인된 3가지 실패 원인

| 실패 조건 | 코드 위치 | 로그 메시지 |
|-----------|-----------|------------|
| JUPITER_API_KEY 미설정 | `preflightCheck.ts:147` | "JUPITER_API_KEY missing" |
| 잔고 부족 | `preflightCheck.ts:153` | "balance 0.0100 SOL < 0.0500 SOL" |
| Edge gate | `measurement.ts:104` | "reject_gate (totalTrades<10)" |

### Bootstrap 역설 (Chicken-and-Egg)

```
live 모드 진입 → edge gate 통과 필요 (totalTrades ≥ 10)
trades 쌓기   → live 모드 필요
→ PREFLIGHT_ENFORCE_GATE=true인 한 영원히 paper
```

**해결: Bootstrap 단계에서는 `PREFLIGHT_ENFORCE_GATE=false` 필수.**
이것이 설계 의도 — enforceGate는 Calibration 이후 활성화하는 게이트.

### Preflight 실패 전체 조건 참조

| 검사 항목 | 기본 임계값 | 코드 위치 |
|-----------|------------|-----------|
| 최소 trades | 50 | `preflightCheck.ts:27` |
| 최소 WR | 40% | `preflightCheck.ts:28` |
| 최소 cost-adjusted R:R | 2.0 | `preflightCheck.ts:29` |
| Jupiter API Key | 필수 | `preflightCheck.ts:147` |
| 지갑 잔고 | ≥ 0.05 SOL | `config.ts:231` |
| Edge gate totalTrades | ≥ 10 | `measurement.ts:104` |
