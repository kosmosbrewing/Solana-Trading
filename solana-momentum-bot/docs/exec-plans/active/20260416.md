### 왜 cupsey는 paper인가

`cupseyLaneHandler.ts`는 live 모드에서 `executeBuy()`를 호출하지만:
- 잔액 이력: 04-11~12 cupsey "+0.52 SOL"이지만 실제 지갑은 1.13→1.04 SOL (감소)
- tx_signature: 27건 전부 null
- slippage: 27건 전부 0 (실거래 불가능한 값)

**결론**: cupsey lane은 paper 모드로 동작했거나, executeBuy가 실패하고 fallthrough로 가격 차이 기반 PnL을 기록하고 있다.

### bootstrap_10s WR 27%의 의미

- 손절:수익 = 73:27. 구조적 음의 기대값.
- MFE(0.23%) < 왕복 비용(0.45%) — 이기는 거래도 비용보다 작다.
- 파라미터 튜닝으로 해결 불가. **이 전략으로는 수익이 불가능하다.**

### 사명 달성 수학

```
현재 잔액: 1.06 SOL
목표:     100 SOL (×94)

0.01 SOL 고정 티켓으로:
  60% WR, avg net +3%/win → E = +0.00012 SOL/trade
  1 SOL → 100 SOL: 19,000 trades 필요 (현실 불가)

1% Kelly 비례 티켓으로:
  60% WR, 승:패 = 3:1 → Kelly 5~10% conservative
  5% 티켓, avg +5% net gain: E = +0.0147 SOL/trade per 0.05 SOL position
  1 SOL → 100 SOL: ~315 trades 필요 (5 trades/day → 63일)
```

**결론**: 사명 달성은 수학적으로 가능하지만, 두 가지가 반드시 필요하다.
1. cupsey 실거래가 실제로 작동할 것
2. 검증된 edge 이후 fixed 0.01 SOL → Kelly 비례 sizing 전환

### Phase 3 (50 trades + WR ≥ 55% 확인 후): Kelly 비례 sizing 전환

**목적**: 0.01 SOL fixed → capital 비례 ticket. 이것이 사명 달성의 핵심.

#### sizing 전환 공식

```
Bootstrap tier  (<20 trades): 0.01 SOL fixed (현재)
Calibration tier (20-49):    0.01 SOL fixed
Confirmed tier  (50-99):     min(capital × 0.01, 0.05) SOL  ← 1% capped at 5%
Proven tier     (100+):      Kelly(WR, avgWin/avgLoss) × 0.3  ← conservative Kelly
```

구현:
```typescript
// cupseyLaneHandler.ts STALK→PROBE 진입 시
function calcTicketSol(capital: number, tradeCount: number, winRate: number, avgRR: number): number {
  if (tradeCount < 50) return 0.01;  // fixed micro-ticket
  if (tradeCount < 100) return Math.min(capital * 0.01, 0.05);  // 1% capped
  const kelly = winRate - (1 - winRate) / avgRR;
  return Math.min(capital * kelly * 0.3, capital * 0.05);  // conservative Kelly
}
```

**Kelly 적용 이전에 필요한 것**:
- `PortfolioTracker` 또는 `EdgeTracker`에서 실시간 capital, winRate, avgRR 집계
- `cupsey_flip_10s` 거래 기록에서 자동 계산
- env override 가능 (`CUPSEY_LANE_TICKET_SOL` 이미 있음, 유지)

### Phase 4 (100+ trades): 복리 성장 경로

```
capital 1.0 SOL, Kelly 5% ticket, WR=60%, avgWin=+5%, avgLoss=-1.5%:
  E per trade = 0.6 × 0.05 × 0.05 - 0.4 × 0.05 × 0.015 = 0.0015 - 0.0003 = +0.0012 SOL
  100 trades: 1.0 × (1 + 0.0012)^100 ≈ 1.13 SOL
  500 trades: 1.0 × (1 + 0.0012)^500 ≈ 1.82 SOL  (시간 단위 아님, 복리임)

실제로는 capital이 커질수록 ticket이 커지는 복리 효과:
  capital=2 SOL → ticket=0.10 SOL → E=+0.0024 SOL/trade
  capital=5 SOL → ticket=0.25 SOL → E=+0.006 SOL/trade
  capital=10 SOL → ticket=0.50 SOL → E=+0.012 SOL/trade

5% Kelly, 60% WR 유지 시: 1 SOL → 100 SOL ~ 300-400 real trades
5 trades/day → 60-80일 이론값
```
