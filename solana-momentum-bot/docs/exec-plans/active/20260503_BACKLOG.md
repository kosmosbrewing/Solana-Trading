Interpretation:

- smart-v3 still has the closest fit to the 5x mission because it has produced large MFE candidates.
- The current live problem is not simply "no signal"; it is a combination of bad early losses, live friction, and exit timing.
- Post-close markouts show that some exits were followed by large continuation, so exit policy needs cohort work before any scale-up.

Latest implementation state:

- smart-v3 live entry no longer uses 24h `independentKolCount` as the live eligibility truth.
- Entry now computes a fresh same-mint KOL context from recent/candidate buys:
  - fresh independent KOL count
  - fresh S/A KOL count
  - fresh burst score
  - last buy age
  - pre-entry sell wave and fresh buys after the last sell
- Velocity live entry is the main live path: fresh 2+ KOLs, fresh S/A strength, fresh score threshold, and recent last buy are required.
- A+A fresh consensus is allowed for live velocity (`KOL_HUNTER_SMART_V3_VELOCITY_SCORE_THRESHOLD=5.0`), while S+B / A+B still fail the fresh S/A strength rule.
- Shadow/inactive KOLs are excluded from live fresh count/score. They are auxiliary only:
  - active 2+ live candidates can carry `SMART_V3_SHADOW_*` confirmation flags.
  - shadow-only candidates can still be measured as paper/shadow observations.
  - active 1 + shadow 1 cannot create live eligibility.
- Pullback remains measurable, but pure pullback live entry falls back to paper by default via `SMART_V3_PULLBACK_LIVE_DISABLED`.
- Weak post-sell recovery falls back to paper via `SMART_V3_POST_SELL_RECOVERY_WEAK` when prior sell flow is not followed by enough fresh independent re-buy.
- Dev wallet status is now a live gate only:
  - `DEV_WALLET_BLACKLIST` / `DEV_WALLET_WATCHLIST` fallback to paper.
  - `DEV_WALLET_ALLOWLIST` is telemetry only and never bypasses security, sell-route, drift, halt, or canary guards.
  - Missing/unknown dev status remains fail-open.
- smart-v3 close rows now also project into:
  - `smart-v3-paper-trades.jsonl`
  - `smart-v3-live-trades.jsonl`
  This is an analysis projection only; `kol-paper-trades.jsonl` / `kol-live-trades.jsonl` remain canonical compatibility ledgers.
- Smart-v3 now has a diagnostic evidence report:
  - command: `npm run kol:smart-v3-evidence-report -- --since 24h --md reports/smart-v3-evidence-YYYY-MM-DD.md --json reports/smart-v3-evidence-YYYY-MM-DD.json`
  - verdicts: `COLLECT`, `DATA_GAP`, `COST_REJECT`, `POST_COST_REJECT`, `WATCH`, `PROMOTION_CANDIDATE`
  - gates: 50/100 close sample thresholds, close-anchor based 30/60/300/1800 buy+sell T+ ok coverage, wallet/token-only/copyable PnL, and post-cost continuation
  - attribution fix: live buy/sell markout anchors now carry `mode`, `armName`, `parameterVersion`, and `entryReason`; unknown-mode markouts no longer fill paper/live cohort coverage.
  - copyable-edge shadow: smart-v3 close/projection rows now carry `smartV3CopyableEdge`, `smartV3CopyablePass`, `smartV3CopyableNetSol`, and required gross move fields.
  - report quality fix: cohort T+ coverage is now `close positionId × anchorType × horizon` based, while row ok-rate is only a secondary quality metric. Closed-trade W/L is copyable/wallet-first; token-only W/L is separate.
  - env: no runtime `.env` change required; `SKIP_SMART_V3_EVIDENCE_REPORT=true` is only a sync-script opt-out.
  - status: report-only; no live entry, exit, or ticket behavior changes.

Latest implementation state:

- When a rotation trigger falls back to paper, the system now runs shadow parameter arms in parallel:
  - `rotation_fast15_v1`: very fast failure detection and small winner capture.

  - entry only for active S/A KOL buys;
  - entry only when our quote is `2%` to `12%` below the KOL's actual weighted fill reference (`solAmount / tokenAmount`);
  - no hot-path RPC fallback is used; missing fill price becomes a no-trade observation;
  - no same-mint KOL sell in the recent window;
  - faster exits: T1 `5%`, trail `2.5%`, profit floor `1.02x`, probe timeout `30s`, hard cut `4%`;
  - too-shallow, too-deep, missing fill price, stale, and recent-sell blocks become no-trade T+ markouts.

- Rotation no-trade probes now preserve token decimals when available, so fresh rejects should stop collapsing into `decimals_unknown`.
- The rotation report now compares arm-level paper closes, post-cost markouts, and rent/network-fee stress.
- Rotation paper visibility now follows the pure_ws pattern: low-noise Telegram digest every 15 minutes plus rare MFE alerts, not per-open/per-close spam.
- Rotation close rows now also project into:

## 4. Decision Gates

### smart-v3 remains main 5x lane if:

- Wallet floor remains intact.
- Live max drawdown does not worsen materially.
- T+ post-close winner-kill cases are understood and reduced.
- Actual MFE distribution continues to show tail candidates.

### rotation-v1 can remain live only if:

- T+15/T+30 post-cost distribution is positive after execution cost.
- Live capture improves, not only paper markout.
- Loss per failed probe is smaller than current behavior.
- It does not interfere with smart-v3 runner preservation.

### pure_ws can graduate from observe-only only if:

- Entry prices are resolved.
